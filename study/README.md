# 量化阶段.
- 今天2026.2.27, 在2026.4.12会有一个算法比赛, 差不多有一个半月的时间(时间期限). 我期望复习算法, 以在算法比赛中能取得好的成绩.
- 目前拥有书籍: 《代码随想录》《代码随想录2图论》. 
- 其中, 《代码随想录2图论》书中有30个具体的算法题目, 目前我已经深入学习了其中的8个, 相较于《代码随想录》而言, 这本书的题目综合性较强, 难度偏大. 而《代码随想录》中, 题目有118道题目, 有部分是简单题, 且部分题目思想已经掌握.
- 打算预留出半个月的时间去刷题, 所以学习上面两本书的截止时间也就是3月28号左右. 
- 每个算法会配套大约20min的讲解视频, 虽然不一定看, 但是也量化一下, 每个算法的输入时间为20min. 那么每个算法输出时间是140min. 一共有158 - 8个算法, 共27000min输出时间, 3000min输入时间, 总待耗时为30000min, 也就是500h. 
- 注意, 3月28号截止是输入的截止时间, 也就是说, 后面半个月我仍然可以输出书中的内容, 并不妨碍我刷算法题. 另外, 上面的总耗时是估算的, 约定算法题目难度统一, 但实际上有很多掌握, 用不了多少时间输出. 对了, 学习理解肯定是每天提升的, 所以总耗时并不会有这么多, 目前来看, 300h的总时间是比较合理的.
- 总共有150道题, 课本学习时间有29天, 差不多每天5道题. 这5道题总输入时间是100min, 每天的单一算法输出时间按照40min来算, 那么每天的总耗时是300min, 也就是5h. 后续如果每天拿出来1h或者1h+的时间用来复习(输出), 那么学习算法的时间总共需要6h. 
- 这样, 29天实际上总学习时长为174h, 后续时间呢, 由于课本内容(输入)已经全部结束了, 所以差不多会拿出来5h时间用来输出课本内容, 大约75h. 这样总消耗时间是250h.
- 这样来看, 时间是在可控范围内的. 但是不要太死板, 比如每天固定5道题. 如果遇到了简单题, 那么就需要往下多看几道. 当然, 遇到难道也要灵活变通.
- 最终来看, 算法相关, 按照书籍, 每天5道题. 其中图论两道, 另一本3道.
